# roadmap/ — visão macro

O que costura o restante: a **direção estratégica** (o que vem e quando, no nível de negócio) e a **execução** (fases de build, sequência e dependências que atravessam os domínios). Dois registros — não confundir os dois sentidos de "Fase": **estratégica** (crescimento do negócio, em `estrategico.md`) × **de build** (construção do app, em `fases.md`).

- [`estrategico.md`](estrategico.md) — registro **estratégico**: ideias validadas (`discovery/`) × horizonte (Agora/Próximo/Depois), respeitando gates e métricas. Hoje a aposta dominante é a **curadoria / segundo funil** (espinha da Fase 1 estratégica — D-26).
- [`fases.md`](fases.md) — registro de **execução**: roadmap faseado de build (Fases 0–4) + **critérios de aceite por fase** (definition of done verificável) + restrições de build conhecidas. É a base de rastreabilidade (`traces`) das WO em [`../tasks/`](../tasks/).
- [`composicao-paginas.md`](composicao-paginas.md) — planta de composição das páginas (home/blog/LP/post) por Bloco; proposta a curar.
- [`inventario-conteudo.md`](inventario-conteudo.md) — tracker das dependências de conteúdo (fotos/vídeos/depoimentos/dados factuais) que gate as fases; `dono`/`prazo` preenchidos pelo fundador.
